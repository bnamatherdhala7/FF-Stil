# Stil
# CLAUDE.md — Read this before every session

## What Stil is

Stil is a conversational AI assistant for creative professionals.
It learns your visual style across sessions and applies it without being asked.
You tell it what to do once — it remembers forever.

Tagline: "Your creative style, remembered."

## The problem Stil solves

Creative professionals — photographers, designers, social media managers,
content creators — make the same editing decisions hundreds of times.
Warm tones. Square crops. Instagram exports. High-contrast backgrounds.

Every existing tool makes you re-specify these preferences every single time.
Lightroom presets are tool-specific. AI assistants are stateless.
Nothing learns your aesthetic and applies it conversationally.

Stil fixes this. It remembers your style profile across sessions and injects
it automatically. A returning user never has to re-explain themselves.

## What Stil does (three capabilities, one system)

### 1. Conversational editing with memory
User describes what they want in natural language. Stil executes editing
actions — filter, crop, brightness, export — and remembers the choices made.
Next session, those preferences are applied automatically.

### 2. Quality insights
Stil tracks whether it's actually serving the user's creative intent — not
just whether it completed the task. Three dimensions measured per session:
- Tool accuracy: did it use the right editing actions?
- Goal completion: did it finish what was asked?
- Creative intent: did the output serve what the user actually wanted?

### 3. Smart asset search
User describes a creative need in plain language. Stil searches their asset
library, tags candidates using AI, and returns ranked recommendations with
rationale. No manual tagging required.

## How the three capabilities connect as one system

```
User types a creative request
          ↓
agent.py loads style_profile.json → injects style into system prompt
          ↓
Claude reasons → calls creative_tools.py functions
(apply_filter, crop_image, set_export_preset, adjust_brightness)
          ↓
Tool results returned → Claude loops until complete → responds
          ↓
Session written to logs/session_TIMESTAMP.jsonl
          ↓
extract_and_save_style() updates style_profile.json
          ↓
Tab "Style":    style_profile.json shown as live profile card
Tab "Insights": insights.py reads logs/ → grades sessions → scorecard
Tab "Assets":   asset_library.py searches assets/ → ranked recommendations
```

Sessions from the chat feed the insights grader.
The style profile updated in chat drives the Style tab display.
Assets searched in the Assets tab are the same files the chat agent references.
One system. Four views.

## What each file does

### app.py — the product UI
Single Streamlit file. Four tabs. All functions imported directly — no
subprocess calls, no separate server processes needed.

Tab "Edit" (main):
  Chat input → run_agent_streaming() → tool pills fire in real time as each
  editing action executes → response streams word by word → style sidebar
  updates after session ends via st.rerun().
  Sample prompts in a selectbox to guide new users.

Tab "Style":
  Reads style_profile.json → shows profile as metric cards: tone, crop
  preference, export platforms. Raw JSON expander. Clear profile button.

Tab "Insights":
  "Run insights" button → calls run_insights() → metric cards per dimension
  (score/5) → bar chart per turn → 3-paragraph quality summary.

Tab "Assets":
  Image grid of assets/ folder → text input for creative brief → calls
  find_asset() → ranked results with thumbnails, tag pills, rationale.

Sidebar: render_sidebar() shows live style_profile.json on every tab.
State: st.session_state for messages, session_log, tool_trace, insights_result.

### agent.py — the core loop
The agentic engine. Decides which tools to call. Manages style memory.

load_style()
  Reads style_profile.json. Returns dict or {} if not yet created.

build_system_prompt(style)
  Base prompt (max 200 tokens) + style injection if profile exists.
  Style injection adds ~100 tokens max.

run_agent_streaming(user_message, style)
  Generator. Yields event dicts consumed by app.py:
    {"type": "tool_start", "tool": name, "input": dict}
    {"type": "tool_end",   "tool": name, "result": dict}
    {"type": "text",       "content": str}
    {"type": "done",       "tool_trace": list}

execute_tool(name, input)
  Dispatches to TOOL_MAP from creative_tools.py. Returns JSON string.

extract_and_save_style(session_log, existing_style)
  One Haiku call. Extracts style_signature JSON from session transcript.
  Merges with existing style_profile.json — never overwrites, updates only.

write_session_log(messages, tool_trace)
  Writes logs/session_TIMESTAMP.jsonl for insights.py to read later.

### creative_tools.py — editing actions
Five functions. No real image processing. Each returns a clean JSON result
confirming what was done. This is what Stil would connect to real editing
tools via MCP in a production version.

  apply_filter(filename, filter_type)    — warm, cool, vintage, dramatic, bw
  adjust_brightness(filename, level)     — -100 to +100
  crop_image(filename, ratio)            — square, landscape, portrait, story
  set_export_preset(filename, platform)  — instagram, print, web, twitter
  list_layers(filename)                  — returns layer stack

Module-level exports used by agent.py:
  TOOL_DEFINITIONS  — Anthropic API tool dicts (name, description, input_schema)
  TOOL_MAP          — {tool_name: function} for execute_tool() dispatch

### asset_library.py — smart asset search (MCP server)
FastMCP server. Four tools. Also importable directly by app.py — no
separate server process needed when running locally.

  list_assets()
    Reads assets/ folder. Returns image filenames.

  inspect_asset(filename)
    Derives metadata from filename keywords: contrast, brightness, tone,
    suitable use cases. No EXIF parsing needed — the filename is the data.

  tag_asset(filename)
    One Haiku call generates 5 descriptive tags. Cached in assets/tags.json.
    Never re-calls the API for the same filename.

  find_asset(prompt)
    Scores each asset against the prompt: filename keyword match ×2,
    tag match ×1, metadata bonus points for contrast/brightness/tone.
    Returns top 3 with rationale explaining why each fits.

### insights.py — quality measurement
Answers: is Stil actually serving creative intent, or just completing tasks?

  load_all_sessions()       reads all logs/*.jsonl
  build_turns(session)      pairs user/agent messages into gradeable turns
  grade_turns_sync(turns)   one Haiku call per turn against 3-dimension rubric
  aggregate_scores()        averages per dimension across all turns
  generate_quality_summary() one Haiku call → 3-para plain-English report
  write_report()            writes insights_report.md
  run_insights()            orchestrates all above, returns dict for app.py

Rubric (1–5 each):
  tool_accuracy:    right editing actions for the request?
  goal_completion:  task completed as asked?
  creative_intent:  did the output serve what the user actually wanted?

creative_intent is the dimension that matters most. A session can score 5 on
goal_completion and 1 on creative_intent — Stil did exactly what was typed but
missed the creative purpose entirely. That distinction is what makes these
insights meaningful, and it's why a rule-based grader wouldn't work here.

## Inputs, state, and outputs

INPUTS:
  Natural language creative requests (Edit tab chat)
  Images in assets/ folder with descriptive filenames (Assets tab)

PERSISTENT STATE:
  style_profile.json    user's style signature across all sessions
  assets/tags.json      cached AI tags per image, never re-generated
  logs/*.jsonl          full session history with tool call traces

OUTPUTS:
  Streaming response with real-time tool execution trace (Edit tab)
  Live style profile card updated after every session (Style tab)
  Quality scorecard with charts and plain-English summary (Insights tab)
  Ranked asset recommendations with thumbnails and rationale (Assets tab)
  insights_report.md written to disk after each insights run

## What makes this agentic

Stil is not a chatbot. The difference:

Chatbot: user types → Claude replies → done. One step, stateless.

Stil: user types → Claude decides which tools to call → executes tool 1 →
observes result → decides next action → loops until complete → saves session
→ extracts style → updates profile for next time.

The test: if you replaced Claude with hardcoded if/else, would it work?
A chatbot: yes. Stil: no — Claude reasons about which tools to use based on
what the user said and what the tools just returned. Remove the reasoning and
the system breaks.

## Real-time UX mechanics

All real-time behaviour flows from run_agent_streaming() in agent.py.
app.py consumes the generator in a for loop:

  tool_start → st.empty() placeholder shows ⚡ firing pill
  tool_end   → replace with ✓ resolved pill + action text
  text       → append to response, update placeholder word by word
  done       → extract_and_save_style() → st.rerun() refreshes sidebar

No JavaScript. No websockets. No threading.

|-------------------------|-------------------|---------------------------|

| Style extraction        | claude-haiku-4-5  | Tiny prompt, simple JSON  |
| Insights grading        | claude-haiku-4-5  | One call per turn         |
| Asset tagging           | claude-haiku-4-5  | Cached after first run    |
| Writing this code       | claude-sonnet-4-6 | Better code quality       |

Never use Opus for runtime tasks.

## Cost guardrails

System prompt: max 200 tokens. Style injection: max 100 tokens.
Tool descriptions: max 40 tokens each. Max turns per session: 6.
Tag cache: never re-tag the same filename.
Target total cost for the entire working prototype: under $1.00.

## Asset naming convention

The filename is the search index. No EXIF, no embeddings, no database.
find_asset() scores by matching prompt words against filenames and cached tags.
Name images so the words in the filename match natural creative briefs:

  portrait_warm_natural_light.jpg      → "warm portrait natural light"
  landscape_cool_tones_wide.jpg        → "cool wide landscape"
  dark_high_contrast_abstract_bg.jpg   → "high contrast background social post"
  bright_minimal_white_desk.jpg        → "clean minimal product shot"
  warm_golden_hour_outdoor.jpg         → "warm golden hour lifestyle"
  dark_moody_studio_portrait.jpg       → "moody dramatic studio portrait"
  bright_colorful_flat_lay.jpg         → "vibrant colorful product flat lay"
  neutral_textured_paper_bg.jpg        → "neutral texture minimal background"
  high_contrast_bw_architecture.jpg    → "black white bold graphic"
  soft_pastel_gradient_bg.jpg          → "soft pastel social media"

Underscores only. No spaces, no special characters.
Free images: unsplash.com — download, rename, drop into assets/.
